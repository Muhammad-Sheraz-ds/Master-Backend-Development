# 1. Introduction

Every program performs two basic operations.

1. Input
2. Output

Input means receiving information from the user or another source.

Output means displaying or storing information.

Examples

Input

- Keyboard
- Mouse
- File
- Network

Output

- Console
- Browser
- File
- Printer

The C++ lecture explains input and output through streams (`cin` and `cout`). In JavaScript, the same ideas exist but with different APIs such as `console`, `prompt()`, and Node.js streams. :contentReference[oaicite:1]{index=1}

# 2. JavaScript Output

The most common output function is

```javascript
console.log();
```

Example

```javascript
console.log("Hello World");
```

Output

```
Hello World
```

# 3. Printing Variables

```javascript
let name = "Ali";

let age = 20;

console.log(name);

console.log(age);
```

Output

```
Ali

20
```

# 4. Printing Multiple Values

```javascript
let name = "Ali";

let cgpa = 3.8;

console.log("Name:", name);

console.log("CGPA:", cgpa);
```

Output

```
Name: Ali
```

# 5. Template Literals

Modern JavaScript provides template literals.

Instead of

```javascript
console.log("Age = " + age);
```

use

```javascript
console.log(`Age = ${age}`);
```

Example

```javascript
let age = 21;

console.log(`Age = ${age}`);
```

Output

```
Age = 21
```

# 6. Console Methods

JavaScript provides several output methods.

## console.log()

General output.

```javascript
console.log("Hello");
```

---

## console.error()

Displays error messages.

```javascript
console.error("File not found.");
```

---

## console.warn()

Displays warnings.

```javascript
console.warn("Low battery.");
```

---

## console.table()

Displays arrays and objects in table form.

```javascript
const students = [

    {name:"Ali", cgpa:3.8},

    {name:"Sara", cgpa:3.9}

];

console.table(students);
```

# 8. Input in Node.js

Node.js uses the **readline** module.

```javascript
const readline = require("readline");

const rl = readline.createInterface({

    input: process.stdin,

    output: process.stdout

});

rl.question("Enter your name: ", function(name){

    console.log("Hello", name);

    rl.close();

});
```

# 9. Reading Numbers

Values entered through `prompt()` or `readline` are strings.

Convert them to numbers.

```javascript
let age = Number(prompt("Age"));
```

or

```javascript
let marks = parseFloat(prompt("Marks"));
```

# 10. Reading Complete Lines

```javascript

```

# 12. Output Formatting

JavaScript does not use manipulators like `setw()` or `setprecision()` from C++. Instead, it provides methods on numbers and strings. :contentReference[oaicite:3]{index=3}

Decimal places

```javascript
let pi = 3.14159265;

console.log(pi.toFixed(2));
```

Output

```
3.14
```

Scientific notation

```javascript
console.log(pi.toExponential(2));
```

Output

```
3.14e+0
```

# 13. Buffering

The C++ lecture explains that output is often buffered and discusses flushing the stream. :contentReference[oaicite:4]{index=4}

In JavaScript:

- Browsers update the console automatically.
- Node.js also buffers output internally.
- Normally, developers do not manually flush the console.

For most JavaScript programs, buffering is handled automatically by the runtime.

# 17. Input/Output Redirection

The C++ lecture concludes with command-line redirection (`>`, `<`, `>>`, `|`). These operators work in the command line for Node.js programs as well. :contentReference[oaicite:5]{index=5}

Examples

Save output to a file

```bash
node app.js > output.txt
```

Read input from a file

```bash
node app.js < input.txt
```

Append output

```bash
node app.js >> log.txt
```

Pipe output

```bash
node app.js | more
```

# Common Mistakes

❌ Forgetting to convert input.

```javascript
let a = prompt("A");

let b = prompt("B");

console.log(a+b);
```

Input

```
10

20
```

Output

```
1020
```

Correct

```javascript
let a = Number(prompt("A"));

let b = Number(prompt("B"));

console.log(a+b);
```

Output

```
30
```

---

❌ Using `==` for numeric comparisons after reading string input.

Convert input to a number before comparing.
