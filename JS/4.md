# 1. Constants

Sometimes a value should never change during program execution.

JavaScript uses the **const** keyword for such values.

Syntax

```javascript
const CONSTANT_NAME = value;
```

Example

```javascript
const PI = 3.141592653589793;

const DAYS_IN_WEEK = 7;

const KM_PER_MILE = 1.60934;
```

Notice:

- Constant names are usually written in CAPITAL letters.
- A constant **must be initialized** when declared.

❌ Wrong

```javascript
const PI;
```

✔ Correct

```javascript
const PI = 3.14159;
```

Using constants makes programs easier to read and avoids **magic numbers**, just as emphasized in the C++ lecture. :contentReference[oaicite:1]{index=1}

# 2. Arrays

An array stores multiple values of the same or different types.

Instead of creating many variables,

❌

```javascript
let mark1 = 80;
let mark2 = 90;
let mark3 = 75;
```

Use an array.

✔

```javascript
let marks = [80,90,75];
```

---

# 3. Array Structure

An array is stored as consecutive elements.

```
Index

0      1      2      3

+-----+-----+-----+-----+

| 80  | 90  | 75  | 95  |

+-----+-----+-----+-----+
```

First element

```javascript
marks[0]
```

Second element

```javascript
marks[1]
```

Arrays begin at index **0**, just like in C++. :contentReference[oaicite:2]{index=2}

# 10. Escape Sequences

Escape sequences allow special characters inside strings.

| Escape | Meaning |
|---------|----------|
| `\n` | New Line |
| `\t` | Tab |
| `\\` | Backslash |
| `\"` | Double Quote |
| `\'` | Single Quote |

Example

```javascript
console.log("Ali\nAhmed");
```

Output

```
Ali

Ahmed
```

Example

```javascript
console.log("Name\tAge");
```

Output

```
Name    Age
```

# 13. Comparison Operators

These operators compare values.

| Operator | Meaning |
|----------|---------|
| > | Greater Than |
| < | Less Than |
| >= | Greater Than or Equal |
| <= | Less Than or Equal |
| == | Equal |
| === | Strict Equal |
| != | Not Equal |

Example

```javascript
console.log(10 > 5);

console.log(10 == "10");

console.log(10 === "10");
```

Output

```
true

true

false
```

Always prefer

```javascript
===
```

because it checks both value and type.

# Prefix vs Postfix

Postfix

```javascript
let x = 5;

console.log(x++);
```

Output

```
5
```

Afterwards

```
x = 6
```

Prefix

```javascript
let y = 5;

console.log(++y);
```

Output

```
6
```

# 17. Operator Precedence

Consider

```javascript
let result = 5 + 2 * 3;
```

What happens first?

```
2 * 3

↓

6

↓

5 + 6

↓

11
```

Multiplication has higher precedence than addition.

Use parentheses when necessary.

```javascript
(5+2)*3
```

Output

```
21
```

The C++ lecture also introduces operator precedence and associativity as essential concepts. :contentReference[oaicite:4]{index=4}

# Common Mistakes

❌ Using `==`

```javascript
10 == "10"
```

Prefer

```javascript
10 === "10"
```

---

❌ Accessing invalid index

```javascript
marks[50]
```

Result

```
undefined
```

---

❌ Forgetting `const`

Wrong

```javascript
PI = 3.14;
```

Correct

```javascript
const PI = 3.14;
```
