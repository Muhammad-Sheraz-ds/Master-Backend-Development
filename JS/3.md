# 1. Introduction

Every programming language follows the same basic principle:

> **Write code → Process the code → Execute the instructions**

In C++, this involves compilation, linking, and creating an executable file. The original lecture illustrates this process with a compilation pipeline and a memory layout diagram. 

JavaScript works differently because it is an **interpreted and Just-In-Time (JIT) compiled** language.

Instead of producing an executable (`.exe`) file, JavaScript code is executed directly by a **JavaScript Engine**.

Examples of JavaScript Engines:

| Platform       | JavaScript Engine |
| -------------- | ----------------- |
| Google Chrome  | V8                |
| Microsoft Edge | V8                |
| Firefox        | SpiderMonkey      |
| Safari         | JavaScriptCore    |
| Node.js        | V8                |

# 2. JavaScript Code Execution

When you write a JavaScript program, several steps occur before your code runs.

```
JavaScript Source Code
        │
        ▼
JavaScript Engine
        │
        ├── Parsing
        ├── Syntax Checking
        ├── JIT Compilation
        ├── Optimization
        ▼
Machine Code
        ▼
Program Execution
```

Unlike C++, JavaScript **does not require**:

* Manual compilation
* Object files
* Linkers
* Executable files

The JavaScript engine handles these tasks automatically.

---

# Example

```javascript
let message = "Welcome to JavaScript";

console.log(message);
```

Execution steps:

1. Read the source code.
2. Check for syntax errors.
3. Convert it into machine instructions.
4. Execute the program.
5. Display the output.

Output:

```
Welcome to JavaScript
```

# 3. Memory in JavaScript

Every program needs memory.

JavaScript automatically allocates and releases memory when needed.

Although developers don't manage memory directly, understanding memory helps explain how variables, arrays, and objects behave.

JavaScript mainly uses two memory areas:

```
+-----------------------+

        STACK

Local Variables
Function Calls
Primitive Values

+-----------------------+

        HEAP

Objects
Arrays
Functions
Classes

+-----------------------+
```

---

# 4. Stack Memory

The **Stack** stores small pieces of data.

Typical contents:

* Local variables
* Function parameters
* Primitive values
* Function call information

Example:

```javascript
let age = 20;

let marks = 80;

let passed = true;
```

These variables are stored efficiently because they contain simple values.

---

# Example

```javascript
let x = 10;

let y = x;
```

Memory

```
Stack

x → 10

y → 10
```

Both variables contain independent copies of the value.

Changing one does not affect the other.

```javascript
y = 50;

console.log(x);
```

Output

```
10
```

---

# 5. Heap Memory

The **Heap** stores complex data structures.

Examples:

* Objects
* Arrays
* Functions
* Classes

Example:

```javascript
let student = {

    name: "Ali",

    cgpa: 3.8

};
```

Memory

```
Stack

student
   │
   ▼

Heap

{

name:"Ali",

cgpa:3.8

}
```

The variable `student` is stored on the stack, but the actual object is stored in the heap.

# 6. Primitive vs Reference Types

JavaScript has two categories of data.

## Primitive Types

* Number
* String
* Boolean
* Undefined
* Null
* BigInt
* Symbol

Primitive values are copied by value.

Example

```javascript
let a = 10;

let b = a;

b = 50;

console.log(a);
```

Output

```
10
```

---

## Reference Types

Reference types include:

* Object
* Array
* Function
* Date
* Map
* Set

Example

```javascript
let person1 = {

    name: "Ali"

};

let person2 = person1;

person2.name = "Sara";

console.log(person1.name);
```

Output

```
Sara
```

Both variables refer to the same object.

# 7. Types of Errors

No programmer writes perfect code on the first attempt.

Errors are a normal part of programming.

The original C++ lecture categorizes errors as **syntax errors, logical errors, warnings, and runtime errors (exceptions)**. 

JavaScript has similar categories.

---

# 8. Syntax Errors

Syntax errors occur when JavaScript grammar rules are violated.

Example

```javascript
let age = ;
```

Output

```
SyntaxError
```

The program cannot run until the syntax is corrected.

---

Another example

```javascript
if(age > 18)

console.log("Adult")
```

Missing braces or parentheses often lead to syntax problems.

---

# 9. Runtime Errors

Runtime errors occur while the program is executing.

Example

```javascript
let user = null;

console.log(user.name);
```

Output

```
TypeError:
Cannot read properties of null
```

Another example

```javascript
let data;

console.log(data.length);
```

Output

```
TypeError
```

---

# 10. Logical Errors

Logical errors are the most difficult to find.

The program runs successfully but produces the wrong result.

Example

```javascript
let total = 90 + 80 + 70;

let average = total;

console.log(average);
```

Output

```
240
```

The correct program should divide by 3.

```javascript
let average = total / 3;
```

Output

```
80
```

# 11. Variables

A variable is a named location that stores data.

Every variable has:

* Name
* Value
* Data Type
* Scope
* Lifetime

Example

```javascript
let age = 20;
```

Here:

| Property | Value  |
| -------- | ------ |
| Name     | age    |
| Value    | 20     |
| Type     | Number |

---

# 12. Declaring Variables

JavaScript provides three keywords.

## let

```javascript
let marks = 80;
```

Use when the value may change.

---

## const

```javascript
const PI = 3.1415926535;
```

Use for values that should never change.

---

## var

```javascript
var total = 100;
```

Older JavaScript code uses `var`.

Modern JavaScript prefers `let` and `const`.

---

# 13. Variable Initialization

Always initialize variables when you create them.

Good

```javascript
let marks = 90;
```

Less desirable

```javascript
let marks;
```

Output

```
undefined
```

Unlike C++, JavaScript does **not** leave variables with random "garbage" values. Instead, an uninitialized variable has the value `undefined`, while the C++ lecture warns that uninitialized variables may contain garbage data. 

---

# 14. Assignment

The assignment operator stores a value inside a variable.

Syntax

```javascript
variable = expression;
```

Example

```javascript
let total = 100;

total = total + 25;
```

Output

```
125
```

---

# 15. JavaScript Data Types

Common primitive data types:

```javascript
let age = 20;          // Number

let name = "Ali";      // String

let passed = true;     // Boolean

let salary;            // Undefined

let account = null;    // Null
```

Unlike C++, JavaScript uses a single **Number** type instead of separate `int`, `float`, and `double`. The engine handles numeric storage internally. 

---

# 16. Scope

Scope determines where a variable can be used.

## Global Scope

```javascript
let university = "ABC University";
```

Accessible from anywhere in the program.

Avoid creating unnecessary global variables.

---

## Function Scope

```javascript
function show(){

    let age = 20;

}
```

`age` exists only inside the function.

---

## Block Scope

```javascript
if(true){

    let marks = 80;

}
```

Outside the block:

```javascript
console.log(marks);
```

Output

```
ReferenceError
```

---

# 17. Lifetime

A variable's **lifetime** is the period during which it exists in memory.

```javascript
function calculate(){

    let sum = 0;

    console.log(sum);

}
```

* `sum` is created when the function starts.
* It is destroyed when the function finishes.

This is similar to the scope and lifetime discussion in the C++ lecture.

# Common Mistakes

❌ Using a variable before declaration

```javascript
console.log(age);

let age = 20;
```

---

❌ Forgetting to initialize a variable

```javascript
let marks;

console.log(marks);
```

Output

```
undefined
```

---

❌ Using `var` in new projects

Prefer:

```javascript
let

const
```
