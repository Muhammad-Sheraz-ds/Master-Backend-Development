# 1. Introduction

Programs do not always execute successfully.

Sometimes unexpected situations occur.

Examples

- Division by zero
- Invalid user input
- Missing files
- Network failures
- Accessing properties of `null`
- Calling undefined functions

These situations are called **exceptions**.

If exceptions are not handled, the program may terminate unexpectedly.

---

# 2. Example Problem

Suppose we write a function to divide two numbers.

```javascript
function divide(num, den) {

    return num / den;

}
```

Calling

```javascript
console.log(divide(20,4));
```

Output

```
5
```

Everything works correctly.

But what happens if

```javascript
divide(20,0);
```

In JavaScript

```
Infinity
```

is returned.

Although JavaScript does not crash like some languages, dividing by zero is usually considered an invalid operation in business applications.

Therefore, we should handle it ourselves.

---

# 3. Checking Exceptional Cases

Always validate data before processing it.

Example

```javascript
function divide(num, den){

    if(den === 0){

        console.log("Division by zero is not allowed.");

        return;

    }

    return num / den;

}
```

Output

```
Division by zero is not allowed.
```

This approach is suitable when the function itself can recover from the problem.

The original C++ lecture emphasizes that exceptional cases should be handled locally when possible. :contentReference[oaicite:2]{index=2}

# 4. Throwing Exceptions

Sometimes a function cannot solve the problem itself.

Instead of silently failing, it should **throw an exception**.

Syntax

```javascript
throw value;
```

Example

```javascript
function divide(num, den){

    if(den === 0){

        throw "Division by zero";

    }

    return num / den;

}
```

The function immediately stops execution.

Control returns to the caller.

This is similar to the `throw` statement shown in the C++ lecture. :contentReference[oaicite:3]{index=3}

---

# 5. Using Error Objects

Although JavaScript allows

```javascript
throw "Error";
```

it is better to throw an **Error object**.

```javascript
throw new Error("Division by zero");
```

Benefits

- Better debugging
- Stack trace
- More meaningful error messages

---

# 6. try...catch

A `try` block contains code that may throw an exception.

A `catch` block handles that exception.

Syntax

```javascript
try{

    // risky code

}
catch(error){

    // handle error

}
```

---

# Example

```javascript
function divide(a,b){

    if(b===0){

        throw new Error("Cannot divide by zero.");

    }

    return a/b;

}

try{

    console.log(divide(20,0));

}
catch(error){

    console.log(error.message);

}
```

Output

```
Cannot divide by zero.
```

This is the JavaScript equivalent of the C++ `try` and `catch` example shown in the lecture. :contentReference[oaicite:4]{index=4}

---

# 7. finally Block

Sometimes certain code should execute whether an exception occurs or not.

Example

```javascript
try{

    console.log("Opening file.");

}
catch(error){

    console.log(error.message);

}
finally{

    console.log("Closing file.");

}
```

Output

```
Opening file.
Closing file.
```

The `finally` block is commonly used to release resources or perform cleanup.

# 8. Creating Custom Errors

```javascript
class AgeError extends Error{

    constructor(message){

        super(message);

        this.name = "AgeError";

    }

}
```

Using the custom error

```javascript
function register(age){

    if(age < 18){

        throw new AgeError("Minimum age is 18.");

    }

    console.log("Registration successful.");

}
```

# 9. What are Function Templates?

The C++ lecture introduces **Function Templates** so that one function can work with multiple data types such as `int`, `float`, or `char`. :contentReference[oaicite:5]{index=5}

JavaScript is different.

Because JavaScript is **dynamically typed**, one function can already accept different types of values.

No special template syntax is required.

---

# 10. Reusable Functions

Example

```javascript
function largest(a,b){

    return a > b ? a : b;

}
```

Integers

```javascript
console.log(largest(10,20));
```

Output

```
20
```

Floating-point numbers

```javascript
console.log(largest(3.5,7.2));
```

Output

```
7.2
```

Strings

```javascript
console.log(largest("Ali","Ahmed"));
```

Output

```
Ali
```

The same function works with multiple types.

---

# 11. Another Example

Array sum

```javascript
function sum(array){

    let total = 0;

    for(const value of array){

        total += value;

    }

    return total;

}
```

Using integers

```javascript
console.log(sum([1,2,3,4]));
```

Output

```
10
```

Using floating-point numbers

```javascript
console.log(sum([1.5,2.2,3.1]));
```

Output

```
6.8
```

Notice that the same function works for both integer and floating-point values without templates.

---

# 12. Why JavaScript Doesn't Need Templates

In C++

```cpp
template<typename T>
T sum(...)
```

is required because every variable has a fixed type.

In JavaScript

```javascript
function sum(array){

    ...

}
```

works naturally for different types because JavaScript determines the type at runtime.

# Common Mistakes

❌ Throwing numbers

```javascript
throw 0;
```

Better

```javascript
throw new Error("Division by zero");
```

---

❌ Ignoring exceptions

```javascript
divide(10,0);
```

Always use

```javascript
try...

catch...
```

when a function may throw an exception.

---

❌ Assuming every value can be added

```javascript
sum([1,"Ali",5]);
```

Result

```
"1Ali5"
```

Always validate input before processing.
